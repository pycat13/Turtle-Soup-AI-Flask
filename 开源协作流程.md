
# 《协作者贡献全流程指南》

>涉及到git的操作你可以使用命令行也可以使用编辑器，本文只给出命令行的操作，编辑器的操作可以自己学一下，更简单一些。
---

# 第 1 步：Fork 仓库

1. 打开原仓库（`https://github.com/yangziyan123/Turtle-Soup-AI-Flask`）
2. 点击右上角 **Fork**
3. 生成属于你自己的仓库：
   `https://github.com/你的用户名/项目名`

> 你的修改会先提交到这个 fork，之后才能向原仓库提交 Pull Request。

---

# 第 2 步：Clone 你自己的 Fork 到本地

在 GitHub fork 页面点击 “Code → HTTPS”，复制地址：

```bash
git clone https://github.com/你的用户名/项目名.git
cd 项目名
```

---

# 第 3 步：添加 upstream（上游仓库，用于同步原作者更新）

拷贝原作者仓库地址：

```bash
git remote add upstream https://github.com/原作者/原仓库名.git
```

检查：

```bash
git remote -v
```

应看到：

```
origin   https://github.com/你的用户名/项目.git
upstream https://github.com/原作者/项目.git
```

---

# 第 4 步：开始开发前 —— 同步 upstream（重要）

每次开发前都要更新：

```bash
git fetch upstream
git checkout main
git merge upstream/main
```

同步完后推到自己的 fork：

```bash
git push origin main
```

---

# 第 5 步：创建一个新的功能分支

不要直接在 main 改！
创建一个 feature 分支：

```bash
git checkout -b feature/你的功能名
```

示例：

```bash
git checkout -b feature/add-login
```

---

# 第 6 步：在该分支上进行开发、修改代码

你现在可以随意开发。

每次修改后提交：

```bash
git add .
git commit -m "描述你做了什么"
```

例如：

```bash
git commit -m "Add login page and user validation"
```

---

# 第 7 步：将你的分支推送到你的 fork

```bash
git push origin feature/你的功能名
```

例如：

```bash
git push origin feature/add-login
```

---

# 第 8 步：在 GitHub 上提交 Pull Request

1. 打开你的 fork 仓库
2. 你会看到提示：**“Compare & pull request”**
3. 点击进入 PR 页面

注意检查：

* **base repository（要合并到）：原作者仓库**
* **base branch（原作者的分支）：main**
* **head repository（你的 fork 仓库）**
* **compare branch（你的 feature 分支）**

确认无误后 → **Create Pull Request**

在描述中写：

* 为什么做这个改动？
* 做了哪些内容？
* 是否影响现有功能？

---

# 第 9 步：等待审查

原作者会：

* 查看你的代码（Review）
* 提出修改建议（Comments）
* 或请求你再修改一次（Request changes）

你可以继续在本地该分支开发，然后：

```bash
git add .
git commit -m "fix review issues"
git push origin feature/xxx
```

> PR 会自动更新，不用重新提。

---

# 第 10 步：等待原作者合并 Pull Request


此时你的代码就正式进入原仓库的 main 分支。

---

# 第 11 步：协作者本地同步 upstream（下次开发前）

```bash
git fetch upstream
git checkout main
git merge upstream/main
git push origin main
```

然后继续创建新的 feature 分支……
